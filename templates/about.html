About the Cheque Fault Detection System

Computer Vision

Uses object detection to locate key fields on cheque images.

Machine Learning

Leverages pre-trained models for field detection and text recognition.

Signature Verification

Compares signatures using structural similarity metrics.

The Cheque Fault Detection System uses a multi-step approach:

  1. Object Detection: A Roboflow-powered model detects key fields on the cheque image, including payee name, amount, MICR code, signature area, and date.
  2. Field Extraction: Detected fields are cropped from the image for further processing.
  3. Quality Assessment: Cheques are classified as "good" if they contain all the required fields (Payee Name, Amount in Numbers, and MICR code).
  4. Optional OCR: The system can extract text from the detected fields using Microsoft's TrOCR model.
  5. Signature Verification: Signature images can be compared using the Structural Similarity Index (SSIM) algorithm.

  • Flask: Python web framework for the application backend
  • Roboflow: Hosting and deployment of object detection models
  • Supervision: Library for visualizing and working with computer vision detections
  • TrOCR: Microsoft's Transformer-based OCR model
  • OpenCV: Image processing operations
  • scikit-image: Advanced image processing functions including SSIM
  • Bootstrap: Frontend CSS framework
  • Matplotlib: For visualizing confusion matrices and metrics

Planned enhancements for the system include:

  • Enhanced signature verification using deep learning-based approaches
  • More sophisticated fraud detection capabilities
  • Integration with banking APIs for automated processing
  • Support for different cheque formats from various countries
  • Improved text extraction quality for handwritten content
  • Batch processing for high-volume applications
